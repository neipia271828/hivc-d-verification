# ローカル実験ログプレビュー設計

## 1. 目的

GPU サーバーで実行した HIVC-D 二者エージェント実験の**完了済みログ**をローカル Mac に取得し、ローカル環境だけで再生・比較できるようにする。

旧設計のように、GPU サーバー上で Web サーバーや ngrok を起動して実験中のログを外部公開する方式は採用しない。

## 2. 背景と変更理由

実験実行・ログ配信・ブラウザ閲覧を GPU サーバーに集約すると、以下が混在する。

- Qwen 推論のための GPU 資源
- 実験途中の不完全な JSONL ストリーム
- ngrok、ポート、HTTP Range リクエストなどの閲覧インフラ
- 実験結果を確認・比較するローカル作業

本設計では、実験と可視化を切り離す。GPU サーバーは再現可能な実験ログを生成するだけとし、ローカル Mac が取得・保存・閲覧を担う。

## 3. スコープ

### 3.1 対象

- GPU サーバー上の `qwen_two_agent_experiment.py` が出力する完了済み実験結果
- 条件別ログ `{condition}_games.csv`
- 全条件結合ログ `all_games.csv`
- 条件別集計 `summary.csv`
- 実験条件、取得日時、取得元を示すローカル側メタデータ

### 3.2 非対象

- 実験実行中のリアルタイム閲覧
- ngrok による公開 URL 発行
- GPU サーバー上の HTTP サーバー常駐
- JSONL をポーリングして途中経過を描画する機能
- ログ取得後に GPU サーバー上の原本を削除・移動する処理

## 4. 利用者フロー

```text
GPU サーバー
  実験を実行
  └─ output_dir/
      ├─ control_games.csv
      ├─ consulting_games.csv
      ├─ hivc_d_games.csv
      ├─ all_games.csv
      └─ summary.csv

ローカル Mac
  1. 取得コマンドを実行
  2. 実験スナップショットをローカルへ保存
  3. ローカルプレビューを起動
  4. 条件、ゲーム、ターンを選び、議論・投票・状態・評価を再生
```

## 5. 保存仕様

### 5.1 ローカル保存先

取得ログは次のディレクトリに、取得単位（run）ごとに保存する。

```text
hivc_sim/results/turn_game/downloads/
└── <run-id>/
    ├── control_games.csv
    ├── consulting_games.csv
    ├── hivc_d_games.csv
    ├── all_games.csv
    ├── summary.csv
    └── manifest.json
```

`run-id` は既定で取得時刻を含む一意な名前とする。利用者は明示的な名前を指定できる。

### 5.2 manifest

`manifest.json` には最低限、次を保存する。

- ローカル取得日時（ISO 8601）
- GPU サーバーのホスト名とリモート実験出力ディレクトリ
- ローカル保存先
- 取得対象ファイル一覧とサイズ
- 取得に使用したツールのバージョン

これにより、表示中の結果がどの GPU 実験から来たものかを追跡可能にする。

## 6. 取得ツール要件

新規の取得コマンドは、`configs/gpu_server.yaml` の SSH 接続情報と `remote_project_dir` を読み取る。

必要な機能は以下とする。

| 機能 | 要件 |
|---|---|
| 取得 | `rsync` を用い、GPU側の実験出力ディレクトリをローカルrunディレクトリへコピーする |
| 原本保護 | GPU側のファイルを変更・削除しない |
| 保存先 | 既定は `hivc_sim/results/turn_game/downloads/<run-id>/` |
| 対象指定 | リモート出力ディレクトリ、run-id、ローカル保存先をCLIで上書き可能にする |
| 検証 | `all_games.csv` または条件別CSV、および `summary.csv` の有無を確認して失敗を明示する |
| 記録 | 成功時のみ manifest を生成する |
| 安全性 | 取得先が既存runを上書きする場合は明示的な許可を要求する |

## 7. ローカルプレビュー要件

プレビューはローカル Mac で起動し、ローカルに取得済みのCSVだけを読む。

| 項目 | 要件 |
|---|---|
| データ選択 | run-id を選択し、そのrun内の `all_games.csv` または条件別CSVを選べる |
| 絞り込み | condition、game_id、turn を選択できる |
| 状態表示 | 行動前後の酸素、電力、船体損傷、浸水、通信、士気、イベント、勝敗を表示する |
| 議論表示 | `discussion_transcript` を発言者・発言目的・本文・必要に応じて思考ログに分けて表示する |
| 決定表示 | alpha/beta票、ready、最終行動、合意またはフォールバック規則、理由を表示する |
| 評価表示 | Q値、best_action、acceptable_actions、regretを表示する |
| 再生 | 前後ターン移動と自動再生を提供する |
| 集計表示 | `summary.csv` の条件別指標を表示する |
| ネットワーク | 起動後の閲覧にGPU接続や外部ネットワークを必要としない |

ローカルHTMLを直接開く方式ではブラウザのファイルアクセス制約が発生しうるため、軽量なローカルHTTPサーバー経由で表示する。公開バインドや ngrok は行わず、既定の待受は `127.0.0.1` とする。

## 8. GPU実験側の変更要件

- `configs/experiment.yaml` の `live_jsonl` は `null` を既定とする。
- GPU実験起動ツールは、ngrok とライブサーバーの起動・停止・URL取得を行わない。
- GPU側の実験は、完了時にCSV群と標準出力ログを残す。
- 実験途中の監視は SSH 上のログ確認を基本とし、閲覧用ストリームを正規経路にしない。

## 9. 旧ライブビューアーの扱い

旧ライブビューアー関連ファイルは、互換性・参照用に次へアーカイブする。

```text
archive/legacy-live-viewer/
└── scripts/
    ├── live_server.py
    ├── visualize_game.html
    └── deploy_viz_to_gpu.sh
```

アーカイブは実行経路・READMEの主手順・GPU運用手順から外す。復元が必要な場合にのみ明示的に参照する。

## 10. 受け入れ基準

1. GPUサーバーにある完了済み実験ログを、1コマンドで新しいローカルrunディレクトリへ取得できる。
2. 取得後にネットワークを切断しても、ローカルプレビューでログを再生できる。
3. `control`、`consulting`、`hivc_d` を同じ画面で切り替え、ゲーム・ターン単位で比較できる。
4. ローカル画面から、議論、投票、採択規則、状態遷移、探索ベース評価を読み取れる。
5. GPUサーバーでngrokやライブサーバーを起動しなくても、実験と結果確認の全手順が完結する。
6. 旧ライブビューアーはアーカイブに残り、新しい通常運用から参照されない。
